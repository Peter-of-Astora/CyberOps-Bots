#### RL Agent Context
The four lower-level RL agents are each specialized to operate within specific assigned subnets. Each agent acts on a single specific node within its subnet per deployment:
(1) Reset Agent: This agent is responsible for resetting a specific node within the subnet. It eliminates the impact of attackers on a compromised node, restoring its normal function/service to an operational state.
(2) Patch Agent: This agent is tasked with improving the security level of a specific node. It reduces the node's probability of being compromised, serving as an effective measure for preventative defense and proactive hardening against attacks.
(3) Isolation Agent: This agent isolates a specific node within the subnet by disconnecting it from other nodes. This action may reduce network availability but is a critical emergency measure to protect overall system security by containing threats.
(4) Recovery Agent: This agent performs the reverse operation of isolation. It restores a previously isolated node by reconnecting it to the network, thereby improving network availability once the threat has been mitigated.
